    • 内存区
      • 方法区
        • 又叫永久区(Perm)
          • -xx:PermSize=128M，默认为物理内存的1/64
          • -XX:MaxPermSize=128M，默认为物理内存的1/4
        • 1.8开始，迁移到MetaSpace
      • 直接内存
        • NIO允许JAVA使用直接内存，不受Xmx限制，适用于读写频繁的场合
      • 堆
        • -Xms128M
        • -Xmx512M
      • Java栈
        • 栈帧
          • 局部变量表
            • 保存参数和局部变量
            • 槽位可复用
              • 栈帧中的变量在出了作用域之后也不会马上弹出，除非后面有新的局部变量复用该槽位
          • 操作数栈
            • 保存计算的中间结果，用于字节码指令的参数传递
          • 帧数据区
            • 保存访问常量池的指针
            • 正常方法返回
            • 异常处理表
          • 非逃逸小对象的栈上分配
        • -Xss128K指定栈大小
      • 本地方法栈
      • PC寄存器
        • 指向当前执行的指令
    • GC
      • 日志读取
        • [GC/[FullGC:垃圾收集的停顿类型，FullGC表示发生了Stop-The-World
        • GC发生的区域
          • 分别为新生代New、老年代Old/Tenured、永久代Perm，名称与GC收集器有关
          • DefNew:Serial收集新生代
          • ParNew:ParNew收集新生的
          • PSYoungGen/PSOldGen/PSPermGen:Paralled Scavenge收集新生代/老年代/永久代
        • 方括号内
          • GC前该内存区域已使用容量->GC后该内存区已使用容量(该内存区总容量)
        • 方括号外
          • GC前Java堆已使用容量->GC后Java堆已使用容量(Java堆总容量)
        • 具体时间
          • user 用户态消耗的CPU时间
          • sys　内核态消耗的CPU时间
          • real 操作从开始到介绍经过的墙钟时间(Wall Clock TIme，包括等待磁盘IO等非计算的等待耗时，CPU时间不含这些)
          • 系统有多CPU或者多核的话，多线程操作会叠加这些CPU时间，所以读者看到user或sys时间超过real时间是完全正常的

    • 组成
      • 概述
        • 由无符号数和表组成
        • 无符号数(u1 u2 u4 u8)
          • 可用于描述数字、索引引用、数量值和UTF-8编码组成的字符串
        • 表
          • 复合结构，以"_info"结尾
      • 文件头部
        • magic(u4)
          • 0xCAFEBABE
        • minor version(u2)
        • major version(u2)
      • 常量池
        • constant_pool_count(u2)
        • constant_pool[contant_pool_count -1](cp_info)
          • 存放字面量literal和符号引用symbolic references
            • 符号引用在类创建或者运行时转换为具体的内存地址
            • 全限定名
            • 简单名称
              • 类和接口中的字段名和方法名使用简单名称存储
            • 描述符
              • 字段描述符
                • B C D F I J(long) S Z(boolean) L(obj reference) [(arr reference)
              • 方法描述符
                • 返回值为void的描述符为Ｖ
          • 从１开始计数，０用于
        • 常量
          • 每一个常量都是一个表
          • 每个表的第一位为u1类型的标志位(14种类型)
      • 类信息
        • access_flags(u2)
        • this_class(u2)
          • 指向CONSTANT_Class_info常量
            • 该常量指向CONSTANT_Utf8-info类型的常量，值为全限定名字符串
        • super_class(u2)
        • interfaces_count(u2)
        • interfaces[interface_count](u2)
      • 字段列表
        • fields_count(u2)
        • fields[fileds_count](field_info)
        • 字段有可能拥有属性，用于存储初始化值、注释等额外信息
      • 方法列表
        • methods_count(u2)
        • methods[methods_count](method_info)
        • 也可以带若干个属性，存储字节码、方法的异常信息等
        • Code属性
          • attribute_name_index永远为指向常量池的索引，值为CONSTANT_Utf8类型的"Code"
          • attribute_length　不含前面６字节
          • 操作数栈深度max_stack
          • 局部变量表最大值max_locals
          • codelength + code[code_length]
          • exception_table_length + exception_table[exception_table_length]
            • start_pc
            • end_pc
            • handler_pc
            • catch_type
            • 如果在start_pc到end_pc偏移量之间遇到catch_type指定的异常，就跳转到handler_pc处执行
          • attributes_count + attributes[attributes_count]
            • 存储行号、局部变量表等信息
            • 比如LineNumberTable LocalVariableTable StackMapTable
            • 除了字段、方法、类文件外，属性本身也可以存储属性
      • 属性表
        • attributes_count(u2)
        • attributes[attributes_count](attribute_info)
          • attribute_info = u2 attribute_name_index + u4 attribute_length + u1 info[attribute_length]
          • Code_attribute
            • 存放方法的字节码指令和其他辅助信息(局部变量表长度、操作数长度、异常表)
            • abastract方法和native方法不含Code属性
